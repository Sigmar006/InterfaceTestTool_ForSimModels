# 总览 — GTest Auto UI 分阶段构建指南

## 四个阶段的文档索引

| 阶段 | 文档 | 核心输入 | 核心输出 |
|---|---|---|---|
| Phase 1 | `Phase1_头文件解析模块.md` | `.h` 头文件 | 函数描述 JSON |
| Phase 2 | `Phase2_代码生成模块.md` | 函数描述 JSON + 用户参数 | CMake 工程目录 |
| Phase 3 | `Phase3_编译执行模块.md` | CMake 工程目录 | 测试结果 JSON |
| Phase 4 | `Phase4_可视化界面层.md` | 以上三个模块 | 完整 Web 应用 |

---

## 推荐给 AI 的执行顺序

每个阶段独立验收后再进入下一阶段。

```
Phase 1 → 验收 → Phase 2 → 验收 → Phase 3 → 验收 → Phase 4
```

Phase 1 和 Phase 2 也可以并行给两个 AI 会话同时构建，因为它们之间没有运行时依赖（只有数据格式依赖，已在文档中明确）。

---

## 各阶段交接数据格式

Phase 1 → Phase 2 的接口 JSON 是整个系统的核心数据契约，格式锁定为：

```json
{
  "schema_version": "1.0",
  "functions": [
    {
      "name": "函数名",
      "namespace": "命名空间或空字符串",
      "return_type": { "raw": "int", "kind": "integer", "is_pointer": false, ... },
      "params": [
        { "name": "参数名", "type": { "raw": "int", "kind": "integer", ... }, "default_value": null }
      ],
      "is_variadic": false,
      "source_line": 12,
      "comment": ""
    }
  ]
}
```

Phase 3 → Phase 4 的结果 JSON 格式锁定为：

```json
{
  "overall_status": "passed|failed|build_error|configure_error|timeout|env_error",
  "summary": { "total": 3, "passed": 2, "failed": 1, "duration_ms": 1200 },
  "stages": {
    "env_check": { "status": "passed", ... },
    "cmake_configure": { "status": "passed", "stdout": "...", "stderr": "..." },
    "build": { "status": "passed", "stdout": "...", "stderr": "..." },
    "test_run": { "status": "passed", "stdout": "..." }
  },
  "test_cases": [
    {
      "id": "AutoTest.my_func_test_001",
      "function_name": "my_func",
      "status": "passed|failed",
      "duration_ms": 1,
      "stdout_captured": "...",
      "failure_message": null
    }
  ]
}
```

## 常见问题与注意事项

### Q: Phase 1 解析时遇到系统头文件路径问题怎么办？
在 `--include` 参数中传入 `/usr/lib/llvm-XX/lib/clang/XX/include`，libclang 需要找到 `stddef.h` 等内建头文件。

### Q: Phase 2 生成的 CMakeLists.txt 里 FetchContent 需要联网，离线环境怎么处理？
使用 `options["use_local_gtest"]` 指定本地已有的 GTest 源码路径，生成的 CMakeLists.txt 会改用 `add_subdirectory` 而非 FetchContent。

### Q: 动态库有依赖其他 .so 的情况怎么处理？
Phase 3 的 `options["env"]` 中设置 `LD_LIBRARY_PATH` 包含所有依赖库的目录，或在 Phase 2 的 CMakeLists.txt 生成时通过 `target_link_directories` 加入。

### Q: C++ 类的成员函数（非静态）能测试吗？
当前版本不支持，Phase 1 的过滤规则中应跳过非静态成员函数。后续版本可考虑在测试代码中插入对象构造逻辑。

### Q: 给 AI 时需要同时提供所有四份文档吗？
不需要，每次只提供当前阶段的文档，避免上下文过长导致 AI 遗漏细节。Phase 4 时可以把前三个阶段已生成的代码文件路径告知 AI，让它直接复用。
